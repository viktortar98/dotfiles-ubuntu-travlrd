# Problem Log Template

Use this as a lightweight durable memory for the audit protocol.

The shape is intentionally light. Extend it only when a repo has a real reason to do so.

```md
# Problem Log

## Entry

- Title:
- Status:
- Recommendation:
- Severity:
- Rationale:
- Where:
- Git commit id:
- Evidence:
- Notes:

Recommended common statuses:
- `new`
- `triaged`
- `in_progress`
- `resolved`
- `deferred`
- `ignored`
```
